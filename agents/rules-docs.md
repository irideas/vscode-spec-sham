# 文档写作与链接规范

## 基础写作规则
- SRS 只写可验证的事实与约束；设计/推荐/反模式放在 SDD 或章节内“建议/演进”小节。
- 交叉引用优先使用绝对路径（如 `/external-entry/uri-and-links-srs`），避免 `../` 相对路径；锚点使用同样策略。
- Tree View 域为深链/外部入口用例的主归宿，平台域（external-entry）引用 Tree 用例而不重复展开消费侧细节；平台域侧重“供给”概念、协议、安全模型与流程。
- 用例编号保持唯一，可在平台域做映射但不重复定义实质内容。
- 文档矩阵格式对齐 Tree View 生态：使用“域 | SRS | SDD | 摘要”四列表头，平台域与 Tree 域保持一致。

## 模板与结构基线
- **Index 页模板**：按顺序包含 (1) 引言/基线（域范围、版本基线）；(2) 角色与阅读路径；(3) 架构/生命周期摘要（3~6 条要点）；(4) 文档矩阵（域 | SRS | SDD | 摘要，链接用绝对路径 `/...`）；(5) 用例导航（用例 ID + 锚点链接）；(6) 质量与实践基线（性能/安全/可用性等）；(7) 怎么用（读者场景）。新增域的 `index.md` 必须对齐此序。
- **SRS 结构模板**：引言（目的/范围/读者/非目标） → 基线或引用 → 总体描述（角色、模型、系统概览、主要流程） → 外部接口与集成约束 → 功能性需求 → 质量/性能/安全/兼容性 → 用例（带 ID 与锚点） → 未来演进/占位。只写可验证事实。
- **SDD 结构模板**：文档背景与目标 → 设计原则 → 使用者关注点或典型场景 → 架构视图（分层/数据流/运行时依赖） → 模块设计 → 模式示例（代码或流程） → 性能/安全/可观测性/测试建议 → 验证与演进路线。SDD 可以包含模式与反模式，但需与 SRS 事实区分。
- **用例与锚点**：用例 ID 在域内唯一，跨域引用使用绝对路径+锚点（例：`/tree-view/tree-view-srs#uc-tree-01`）；平台域可映射但不重复定义内容。
